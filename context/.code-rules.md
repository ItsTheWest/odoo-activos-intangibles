# Code Rules

This document outlines the coding standards and best practices for the `gestion_activos_intangibles` module.

## 1. Odoo Standards
- Follow standard Odoo guidelines (OCA guidelines are preferred).
- All models must inherit from `models.Model` or `models.TransientModel`.
- Use descriptive naming conventions for fields and methods.

## 2. Security
- Define proper access rights in `security/ir.model.access.csv`.
- Implement record rules if data access needs to be rest ricted by user or company.

## 3. Views
- Use standard Odoo XML view structures (Tree, Form, Kanban).

## 4. Professional Development Workflow & Contribution Strategy
- **Module Milestone (Balanced Atomic Commits)**: Perform a git commit upon completing a logical unit within a single module. Bundle related files (e.g., view, logic, data) into a single professional commit.
- **Scope Limiting**: If changes become too extensive (e.g., >10-15 files), divide them into smaller logical commits.
- **Prohibition of Multi-Module Bundling**: Never group changes from unrelated modules or distinct architectural layers into a single commit.
- **Context Synchronization**: Adhere to `context/` guidelines for commit messages and code structure.
- **Language**: Technical documentation and commit messages MUST be in English.
